HR-RAG

Retrieval-Augmented AI for Automated HR Reports

RAG LLM HR Automation

Overview

HR-RAG is a specialized Retrieval-Augmented Generation (RAG) system designed to automate the creation of professional legal and HR strategy reports (e.g., Response Strategy for the "Yellow Envelope Law").

It integrates Upstage Document AI for high-fidelity parsing, Solar Pro/GPT-4o LLMs for reasoning, and a robust Hybrid Search (Vector + BM25) pipeline with Reranking to deliver highly accurate and context-aware outputs.

System Architecture

  • Parsing (Document AI): Uses Upstage Layout Analysis to preserve complex table structures and headers, ensuring no semantic loss during extraction.
  • Indexing (Hybrid): Combines Solar Embedding (Semantic) and BM25 (Keyword/Entity) to capture both nuance and exact matches.
  • Retrieval & Generation: Hybrid Search (0.7 Vector + 0.3 BM25) retrieves candidates, followed by BGE-M3 Reranking to refine the Top-60 results into the precise Top-5 for the LLM.

Performance & Optimization

  • Precision Boost: Implementation of the BGE-M3 Reranker significantly improved Top-5 context relevance.
  • Efficiency: Incremental Indexing logic automatically detects processed files, reducing API costs and boosting execution speed by over 10x.
  • Validation: Proven effectiveness in generating detailed compliance reports from complex PDF inputs.

Future Roadmap

  • Section-based Chunking: Utilizing layout header hierarchy for better context preservation (vs. simple character splitting).
  • Metadata Filtering: Enabling filtered search by document year, type (Case Law vs. Statute), and jurisdiction.
  • Query Expansion: Implementing advanced query rewriting to handle ambiguous user intents.
"From precise table parsing to hybrid retrieval — HR-RAG transforms static legal documents into dynamic, actionable insights."
